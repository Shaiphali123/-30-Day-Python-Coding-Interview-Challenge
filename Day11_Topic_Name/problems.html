Sample Test Cases

Test Case 1:

Input: arr = [1, 2, 4, 7, 11, 15], target = 15
Output: True
Explanation: 4 + 11 = 15 (pair found at indices 2 and 4)

Test Case 2:

Input: arr = [1, 2, 3, 4, 5], target = 10
Output: False
Explanation: No two numbers in the array sum to 10

Test Case 3:

Input: arr = [-1, 0, 1, 2, 3], target = 2
Output: True
Explanation: -1 + 3 = 2 or 0 + 2 = 2 (multiple valid pairs exist)

Test Case 4 (Edge Case):

Input: arr = [1], target = 1
Output: False
Explanation: Only one element, cannot form a pair

Expected Approach

Use the Two Pointer Approach to solve this efficiently:

  • Time Complexity: O(n)
  • Space Complexity: O(1)
  • Take advantage of the sorted array property

Hint 1: Think about pointer placement

Place one pointer at the beginning and one at the end of the array. This gives you the smallest and largest available values.

Hint 2: Consider the sum comparison

If current sum is less than target, you need a larger sum (move left pointer right). If current sum is greater than target, you need a smaller sum (move right pointer left).

Hint 3: Think about the termination condition

The algorithm terminates when pointers meet (left >= right) or when you find the target pair.
